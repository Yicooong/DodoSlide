# 新闻播报设计方法论 (News Broadcast SOP)

---

## 第一阶段：头条提炼

新闻播报的核心是"信息层级"。面对用户输入，先提炼新闻价值：

| 内容类型 | 新闻化策略 | 视觉表达 |
|---------|-----------|---------|
| 核心观点 | 提炼为头条标题 | 超大字号 + 深色底 + 红色标签 |
| 数据指标 | 转化为"突发数据" | 大数字 + 实时标签 + 背景图 |
| 流程/步骤 | 转化为"事件时间线" | 时间轴 + 节点标注 |
| 对比/比较 | 转化为"正反方观点" | 双栏 + 红/蓝标签 |
| 技术细节 | 转化为"深度报道" | 图文并排 + 详细说明 |
| 图片展示 | 转化为"现场画面" | 大图 + 底部滚动字幕 |
| 引用/金句 | 转化为"当事人发言" | 引用框 + 人名标注 |
| 章节过渡 | 转为"新闻提要" | 全屏深色 + 标题预览 |

**核心原则：一切内容都是"新闻"。用新闻编辑室的视角重新包装信息。**

---

## 第二阶段：播报布局

### 信息层级
新闻播报的布局遵循严格的视觉层级：头条 > 副标题 > 正文 > 来源。

### 布局选择

| 布局模式 | 适用场景 | 实现要点 |
|---------|---------|---------|
| 头条封面 | 核心观点、开场 | 超大标题 + 深色底 + 红色标签 |
| 新闻速报 | 数据、指标 | 大数字 + 实时标签 + 背景图 |
| 深度报道 | 详细分析 | 左图右文或左文右图 |
| 专家访谈 | 引用、观点 | 引用框 + 人名 + 头衔 |
| 数据看板 | 多维数据 | 网格数据卡 + 红色强调 |
| 时间线报道 | 流程、历史 | 水平时间轴 + 节点 |
| 新闻提要 | 章节过渡 | 全屏深色 + 下期预告 |
| 现场画面 | 图片展示 | 大图 + 底部滚动字幕条 |

### 滚动字幕条
- 固定在页面底部，高度 ~6vh
- 深色背景（`#1A1A2E`）
- 白色等宽字体
- 包含新闻来源/时间/关键词

---

## 第三阶段：紧迫感色彩

### 配色方案

| 色彩角色 | 色值 | 使用场景 |
|---------|------|---------|
| 主背景 | `#FFFFFF`（纯白） | 正文页背景 |
| 深色背景 | `#1A1A2E`（深夜蓝） | 头条封面、新闻提要 |
| 头条红 | `#DC2626`（新闻红） | 标签、强调、分割线 |
| 主文字 | `#111827`（墨黑） | 标题、正文 |
| 辅助文字 | `#6B7280`（中灰） | 来源、时间、注释 |
| 强调蓝 | `#2563EB`（信息蓝） | 链接、次要强调 |
| 警告黄 | `#F59E0B`（警示黄） | 警告、突发标识 |

### 标签系统
- "BREAKING" / "突发"：红色背景 + 白色文字 + 大写字母
- "LIVE" / "直播"：红色背景 + 白色文字 + 圆点动画
- "EXCLUSIVE" / "独家"：深蓝背景 + 白色文字
- "UPDATE" / "更新"：黄色背景 + 黑色文字
- 标签位置：标题左上方或左上角

## 第四阶段：播报排版

### 字体选择

| 元素 | 字体 | 字重 | 字号参考 |
|------|------|------|---------|
| 头条标题 | 无衬线体（font-sans） | 800-900（ExtraBold/Black） | 4vw-7vw |
| 副标题 | 无衬线体（font-sans） | 600（SemiBold） | 2vw-3vw |
| 正文 | 无衬线体（font-sans） | 400（Regular） | 1vw-1.2vw |
| 滚动字幕 | 等宽体（font-mono） | 400（Regular） | 0.8vw |
| 数据数字 | 无衬线体（font-sans） | 700（Bold） | 6vw-12vw |
| 来源标注 | 无衬线体（font-sans） | 400（Regular） | 0.7vw |

**关键规则：标题必须加粗且醒目。新闻风的力量感来自字重和字号的双重冲击。**

### 标题处理
- 头条标题全大写（`uppercase`）或中英文混排时英文大写
- 标题行间距紧凑（`leading-tight`）
- 标题下方添加红色分割线（3px）
- 副标题在标题下方，间距 `mt-3`

### 来源标注
- 格式：`来源：XXX | 时间：YYYY-MM-DD`
- 位置：正文下方或页面底部
- 颜色：辅助灰 `#6B7280`
- 字号：小号（0.7vw-0.8vw）

## 第五阶段：元素规范

### 容器样式
- 新闻卡片：`bg-white border border-[#E5E7EB]`，直角，无阴影
- 头条容器：`bg-[#1A1A2E]`，全宽，带红色顶部边框
- 引用框：左侧红色竖线 `border-l-4 border-[#DC2626]`
- 数据卡：`bg-[#F9FAFB] border border-[#E5E7EB]`，直角

### 允许的元素
- 文字（无衬线为主）
- 红色标签/徽章
- 分割线（1px-3px）
- 滚动字幕条
- 图片（带来源标注）
- lucide-react 图标（功能性标注）
- 时间戳

### 禁止的元素
- 圆角（`rounded-*`）
- 阴影（`shadow-*`）
- 渐变背景
- emoji
- 装饰性图案
- 柔和色调（保持高对比度）

### 图片处理
- 图片占满容器宽度
- 图片底部叠加半透明黑色条，显示来源/说明
- 图片比例统一（16:9）
- 图片可加轻微暗角效果（`vignette`）

---

## 第六阶段：可信度自检

### P0（必须通过）
- 画布尺寸正确：`w-[1280px] h-[720px]`（16:9）或 `w-[1024px] h-[768px]`（4:3）
- 新闻红（`#DC2626`）作为主强调色
- 标题字重 800 以上，足够醒目
- 至少有 1 个红色标签或分割线
- 无圆角、无阴影、无渐变
- 使用 lucide-react 图标，不使用 emoji

### P1（播报节奏）
- 至少使用 3 种不同布局模式
- 不连续 3 页使用相同布局
- 深色封面/提要页与白色内容页交替
- 8 页以上 deck 至少有 1 个头条封面 + 1 个新闻提要

### P2（细节打磨）
- 所有数据标注来源（"来源：XXX"）
- 时间戳格式统一（YYYY-MM-DD 或 HH:MM）
- 滚动字幕条存在且内容相关
- 红色标签使用一致（不混用多种标签风格）
- 标题到正文间距足够（`mt-4` 到 `mt-6`）
- 文字对比度达标（墨黑 `#111827` on `#FFFFFF`）
