# 蓝图工程设计方法论 (Blueprint Engineering SOP)

---

## 第一阶段：系统分析

蓝图风格的核心是"工程制图"。面对用户输入，先解构为系统层级：

| 内容类型 | 工程化策略 | 视觉表达 |
|---------|-----------|---------|
| 系统架构 | 分层架构图 | 层级堆叠 + 连接线 |
| API/接口 | 端点文档卡 | 参数表格 + 方法标签 |
| 数据流 | 流程图 | 节点 + 箭头 + 标注 |
| 技术指标 | 仪表盘面板 | 网格数据卡 + 单位标注 |
| 对比/选型 | 技术评估矩阵 | 表格 + 评分标注 |
| 部署/运维 | 拓扑图 | 节点连接 + 状态标识 |
| 时间线 | 里程碑图 | 水平轴 + 节点标注 |
| 代码/配置 | 代码片段 | 等宽字体 + 行号 |

**核心原则：一切内容都是"技术图纸"。用工程师的视角重新组织信息。**

---

## 第二阶段：分层架构布局

### 网格系统
蓝图风格使用严格的 12 列网格，所有元素必须对齐到网格线。

### 布局选择

| 布局模式 | 适用场景 | 实现要点 |
|---------|---------|---------|
| 架构分层图 | 系统设计 | 上下分层，每层占满宽度，层间连接线 |
| 双栏文档 | API 文档、对比 | 左 40% 说明 + 右 60% 图表/代码 |
| 三栏仪表盘 | 指标展示 | 三列等宽数据卡 |
| 全宽流程图 | 数据流、流水线 | 横向节点 + 箭头连接 |
| 代码面板 | 配置、代码 | 单列，带行号和语法高亮 |
| 网格矩阵 | 多维对比 | 2x2 或 3x2 等分网格 |

### 层级关系表达
- 上层为抽象层，下层为实现层
- 层间用 `border-b` 分割线分隔
- 连接线使用 `border-l` 或 SVG 线条
- 层级标签使用小号等宽字体 + 大写字母

## 第三阶段：蓝图色彩系统

### 配色方案

| 色彩角色 | 色值 | 使用场景 |
|---------|------|---------|
| 图纸底色 | `#F8F9FA`（冷白） | 主背景 |
| 线条色 | `#4A5568`（钢灰） | 边框、分割线、连接线 |
| 标注色 | `#2D3748`（墨灰） | 文字、标签 |
| 强调色 | `#3182CE`（蓝图蓝） | 关键数据、状态标识、连接线 |
| 警告色 | `#E53E3E`（警告红） | 错误、风险标识 |
| 成功色 | `#38A169`（通过绿） | 成功、在线状态 |
| 辅助色 | `#A0AEC0`（银灰） | 次要信息、网格线 |

### 线条系统
- 主边框：`border-[#4A5568]`，1px 实线
- 次要分割线：`border-[#A0AEC0]`，1px 实线
- 虚线连接：`border-dashed`，用于间接关系
- 网格背景线：使用 CSS `background-image: linear-gradient()` 创建淡网格

### 背景网格
- 网格间距：20px
- 网格线颜色：`rgba(74, 85, 104, 0.08)`（极淡钢灰）
- 网格仅在内容页使用，封面/收尾页可省略

## 第四阶段：工程排版

### 字体选择

| 元素 | 字体 | 字重 | 字号参考 |
|------|------|------|---------|
| 系统标题 | 无衬线体（font-sans） | 600（SemiBold） | 2.5vw-4vw |
| 层级标签 | 等宽体（font-mono） | 600（SemiBold） | 0.8vw |
| 正文说明 | 无衬线体（font-sans） | 400（Regular） | 1vw-1.1vw |
| 数据数字 | 等宽体（font-mono） | 700（Bold） | 4vw-8vw |
| 代码/参数 | 等宽体（font-mono） | 400（Regular） | 0.85vw |
| 标注文字 | 等宽体（font-mono） | 400（Regular） | 0.7vw |

### 标注系统
- 层级标签：`A.` `B.` `C.` 或 `L1` `L2` `L3` 前缀
- 组件标签：等宽体 + 小字号 + 钢灰色
- 参数标注：`key: value` 格式，冒号对齐
- 单位标注：必须标明单位（ms、req/s、MB 等）

### 对齐规则
- 所有内容严格左对齐
- 多列内容顶部对齐（`items-start`）
- 表格列内容居中对齐
- 代码块保留缩进

## 第五阶段：组件规范

### 容器样式
- 卡片：`bg-white border border-[#4A5568]`，直角，无阴影
- 代码块：`bg-[#F7FAFC] border border-[#A0AEC0]`，等宽字体
- 状态标签：小字号，带左侧色块标识（绿/红/黄）
- 层级容器：底部 1px 实线分隔，左侧标签

### 允许的元素
- 文字（无衬线 + 等宽混用）
- 边框容器（1px 实线为主）
- 连接线（实线/虚线）
- 网格背景
- 表格
- lucide-react 图标（功能性标注）
- 箭头/指向符号

### 禁止的元素
- 圆角（`rounded-*`）
- 阴影（`shadow-*`）
- 渐变背景
- emoji
- 装饰性图案
- 暖色调

### 图片/图表处理
- 图片加钢灰色边框
- 图片比例统一（16:9 或 4:3）
- 图表使用蓝图蓝色系
- 图表标注使用等宽字体

## 第六阶段：技术精度自检

### P0（必须通过）
- 画布尺寸正确：`w-[1280px] h-[720px]`（16:9）或 `w-[1024px] h-[768px]`（4:3）
- 蓝图蓝色（`#3182CE`）作为唯一强调色
- 所有边框为 1px 实线，直角
- 等宽字体用于所有技术标注
- 无圆角、无阴影、无渐变
- 使用 lucide-react 图标，不使用 emoji

### P1（工程节奏）
- 至少使用 3 种不同布局模式
- 不连续 3 页使用相同布局
- 封面/收尾页与内容页有明显区分
- 8 页以上 deck 至少有 1 个架构图 + 1 个数据面板

### P2（精度打磨）
- 网格背景存在但不喧宾夺主（透明度 < 10%）
- 标注文字对齐整齐（冒号对齐、列对齐）
- 连接线有明确方向（箭头标识）
- 数据数字标明单位（ms、%、MB 等）
- 层级关系清晰（上下分层，标签明确）
- 文字对比度达标（墨灰 `#2D3748` on `#F8F9FA`）
