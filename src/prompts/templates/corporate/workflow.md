# 企业商务设计方法论 (Corporate Clean SOP)

---

## 第一阶段：内容语义解析

企业商务风强调数据驱动与逻辑严谨。面对用户输入，先识别内容类型并映射到对应布局：

| 内容类型 | 视觉策略 | 说明 |
|---------|---------|------|
| 财务数据/指标 | KPI Grid 或 Data Table | 数字优先，表格化呈现 |
| 战略概览 | Two Column Split | 问题+方案，数据+分析 |
| 团队介绍 | Feature Cards（3列网格） | 头像+姓名+职位，卡片式 |
| 流程/步骤 | Process Steps | 编号圆圈+连接线+描述 |
| 市场分析 | Data Table + Two Column | 数据表格配合文字解读 |
| 产品特性 | Feature Cards | 左侧 accent 竖线卡片 |
| 时间线/里程碑 | Timeline | 水平圆点+连接线+描述 |
| 总结/收尾 | Summary Closing | 海军蓝底+白字+要点列表 |

**核心原则：数据先行，结论后置。** 每页幻灯片应先呈现数据/事实，再给出解读。

---

## 第二阶段：布局决策矩阵

### 布局选择逻辑

根据内容特征选择布局，遵循以下决策矩阵：

| 内容特征 | 推荐布局 | 网格结构 | 适用场景 |
|---------|---------|---------|---------|
| 财务数据（3个以上指标） | KPI Grid | `grid grid-cols-3 gap-4` | 季度报告、业绩展示 |
| 财务数据（表格形式） | Data Table | `w-full` 表格 | 收入明细、预算对比 |
| 战略概览（对比型） | Two Column Split | `grid grid-cols-2 gap-8` | 问题vs方案、现状vs目标 |
| 团队/部门介绍 | Feature Cards | `grid grid-cols-3 gap-4` | 组织架构、核心团队 |
| 流程/方法论 | Process Steps | `flex items-center gap-0` | 实施路径、工作流程 |
| 单一核心数据 | KPI Grid（单卡） | 居中大数字 | 年度总收入、用户总数 |

### 网格规范
- 正文页默认使用 `grid grid-cols-12 gap-4` 基础网格
- 通过 `col-span-N` 控制内容比例
- 信息密度高时：大 col-span（8 或 9）给文字区
- 并列对比时：1:1（6+6）或三等分（4+4+4）

### 标题规范
- 标题默认左对齐，位于页面左上内容轴
- 禁止将正文标题居中（封面/收尾页除外）
- Kicker（分类标签）必须在标题上方，不和标题同行

## 第三阶段：内容结构化

### 问题-方案-请求框架

企业演示的核心叙事结构：

1. **问题定义**（Problem）：用数据量化痛点
   - 布局：Two Column Split，左栏放问题数据
   - 关键：数字要大、要醒目（`text-4xl font-bold`）

2. **方案阐述**（Solution）：用流程/特性展示解决方案
   - 布局：Process Steps 或 Feature Cards
   - 关键：逻辑链条清晰，步骤编号明确

3. **资源请求**（Ask）：明确需要什么
   - 布局：KPI Grid 展示预算/资源需求
   - 关键：具体数字，不要模糊描述

### 数据呈现规范
- 所有数据必须有单位和时间范围
- 同比/环比变化用 Status Badge 标注（绿色增长/红色下降）
- 表格必须有表头（海军蓝底白字），行交替灰白底
- KPI 卡片必须包含：标签、数值、变化说明

## 第四阶段：视觉克制

### 配色纪律
- 主色仅限海军蓝 `#0A2540`，辅色仅限蓝 `#1D4ED8`
- 强调色只用于：关键数据、按钮、accent 线、状态标签
- 禁止大面积使用强调色——大面积色块仅在封面/收尾页

### 装饰最小化
- 唯一允许的装饰：`w-12 h-0.5 bg-[#0A2540]` 水平 accent 线
- 分割线只用 1px hairline：`border-t border-[rgba(10,37,64,0.12)]`
- 卡片边框统一：`border border-[rgba(10,37,64,0.12)] rounded-md`
- 禁止渐变、大圆角、装饰性几何元素

### 阴影控制
- 仅允许 `shadow-sm` 或无阴影
- 禁止 `shadow-md` 以上的阴影效果

---

## 第五阶段：质量自检

### P0（必须通过）
- 画布尺寸正确：`w-[1280px] h-[720px]`
- 配色一致：主色=海军蓝 `#0A2540`，辅色=蓝 `#1D4ED8`
- 使用 lucide-react 图标，不使用 emoji
- 圆角控制在 `rounded` 到 `rounded-md`
- 标题左对齐，正文标题不居中
- Kicker 在标题上方（上下结构）
- 禁止渐变、禁止自定义 hex 值

### P1（布局节奏）
- 使用了至少 3 种不同的布局模式
- 不连续 3 页使用同一布局
- 封面和收尾页使用海军蓝底
- 8 页以上至少有 1 个封面 + 1 个收尾
- 数据页有清晰的表头和交替行色

### P2（视觉打磨）
- KPI 数字字号足够大（至少 `text-4xl`）
- 表格对齐规整，列宽合理
- 卡片间距统一（`gap-4` 或 `gap-5`）
- 所有阴影为 `shadow-sm` 或无阴影
- Status Badge 颜色语义正确（绿=正向，红=负向）
