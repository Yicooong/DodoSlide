# 杂志风设计方法论 (Magazine Design SOP)

---

## 第一阶段：内容语义解析

面对用户输入，先将内容归入以下四种"元范式"，每种对应不同的视觉策略：

| 范式 | 核心逻辑 | 视觉策略 |
|------|---------|---------|
| Problem（问题） | 现状、痛点、约束 | → 挑战卡片，深色底，突出矛盾 |
| Method（方法） | 流程、架构、步骤 | → Pipeline 流水线或层级堆叠 |
| Result（结果） | 数据、对比、指标 | → Big Numbers 大字报或图表 |
| Discussion（讨论） | 归因、展望、引用 | → 大引用页或 Callout 引用框 |

---

## 第二阶段：布局决策矩阵

根据识别出的内容语义，自动选择布局：

| 内容语义 | 触发信号 | 推荐布局 |
|---------|---------|---------|
| 单一核心声明 | 只有一句话/一个结论 | Hero Cover 或 Hero Question |
| 多维并列 | 列举多个同级项 | Big Numbers Grid 或 Grid |
| 逻辑递进 | "首先...然后...最后" | Pipeline 流水线 |
| 对比辩证 | "优化前 vs 优化后" | Before/After |
| 深度解析 | 复杂的完整闭环 | Quote + Image（左文右图） |
| 大段信息 | 信息密度高 | Lead Image + Side Text |
| 金句/观点 | 关键引用 | Big Quote |
| 章节过渡 | 幕间切换 | Act Divider |
| 多图展示 | 截图/照片集合 | Image Grid |
| 悬念收束 | 留下思考 | Hero Question |

## 第三阶段：主题节奏规划

**在写任何代码之前，先列出每一页的主题分配。**

### 深/浅交替规则
- Hero 页（封面、幕封、问题页）：深色底，仪式感
- 正文页（数据、图文、流水线）：深浅交替，每 2-3 页切换
- 大引用页：优先深色底，金句仪式感
- 禁止连续 3 页以上相同背景色

### 8 页节奏模板（可直接套用）

| 页 | 背景 | 布局 | 备注 |
|---|---|---|---|
| 1 | 深色 | Hero Cover | 开场 |
| 2 | 浅色 | Big Numbers | 数据抛出 |
| 3 | 深色 | Quote + Image | 对比/故事 |
| 4 | 浅色 | Pipeline | 流程 |
| 5 | 浅色 | Act Divider | 呼吸 |
| 6 | 深色 | Quote + Image 或 Big Quote | |
| 7 | 深色 | Hero Question | 悬念收束 |
| 8 | 浅色 | Big Quote / 结尾 | 收尾 |

## 第四阶段：视觉规范

### 配色约束（6-3-1 原则）
- 60% 背景色（纸色）
- 30% 骨架色（墨色文字、边框）
- 10% 强调色（极少用，杂志风的克制感来自少用强调色）
- 整体以中性暖色为主，避免高纯度颜色

### 排版约束
- 标题用 serif（font-serif），正文用 sans（font-sans），元数据用 mono（font-mono）
- 标题字重 700-900，正文字重 400
- 行间距给足：正文用 `leading-relaxed`，标题用 `leading-tight`
- Kicker（引导句）必须在标题上方，不能和标题同行

### 间距约束
- 页面外边距：`p-10` 或 `px-12 py-8`
- 卡片内边距：`p-6`
- 标题到内容间距：`mt-6` 到 `mt-8`
- 标题区与正文区之间要有明显的"空气感"

### 边框约束
- 减少实线，多用微妙的灰度差区分区域
- 如果用边框，用 `border-[#E8E5DE]`（纸色浅）
- 引用框用 `border-l-4` 左竖线，不用全框

---

## 第五阶段：质量自检

### P0（必须通过）
- 画布尺寸正确：`w-[1280px] h-[720px]`（16:9）或 `w-[1024px] h-[768px]`（4:3）
- 深浅交替：不连续 3 页同色
- Kicker 在标题上方（上下结构），不和标题同行
- 图片用标准比例，不用原图奇葩比例
- 图片容器直角，不加圆角/阴影
- 使用 lucide-react 图标，不使用 emoji

### P1（布局节奏）
- Hero 页和正文页交替（每 3-4 页插入 1 个 Hero）
- 8 页以上的 deck 至少有 1 个深色 Hero + 1 个浅色 Hero
- 不能整个 deck 只有浅色正文页

### P2（视觉打磨）
- 左文右图时，图片与正文内容对齐（不和超大标题顶端齐平）
- 标题到正文之间有明显间距
- Chrome 页眉和 Kicker 内容不重复（Chrome 是栏目标签，Kicker 是本页钩子）
