# 瑞士风设计方法论 (Swiss Design SOP)

---

## 第一阶段：内容语义解析

瑞士风强调结构理性。面对用户输入，先识别内容类型：

| 内容类型 | 视觉策略 |
|---------|---------|
| 数据/指标 | KPI Tower 或 H-Bar Chart |
| 概念并列 | Three Forces Cards 或 Four Cards |
| 对比/比较 | Duo Compare 或 Split Statement |
| 流程/步骤 | Timeline 或 Pipeline |
| 声明/观点 | Dot Matrix Statement 或 Index Cover |
| 图片展示 | Image Hero 或 Matrix 网格 |
| 系统/架构 | System Diagram（grid 堆叠） |
| 时间序列 | Timeline Walk |

---

## 第二阶段：布局决策

### 12 列网格优先
所有正文页默认使用 `grid grid-cols-12 gap-4`，通过 `col-span-N` 控制比例。

### 布局选择原则
- 信息密度高 → 大 col-span（8 或 9）给文字区
- 视觉主导 → 大 col-span 给图片/图表
- 并列对比 → 1:1（6+6）或三等分（4+4+4）
- 单一焦点 → 全宽 12 列，居中内容

### 标题位置
- 标题默认左对齐，位于页面左上内容轴
- 禁止将正文标题居中
- Kicker（mono 标签）必须在标题上方（上下结构），不和标题同行

---

## 第三阶段：字重层次

**这是瑞士风最重要的规则：文字越大，字重越轻。**

| 元素 | 字重 | 原因 |
|------|------|------|
| Hero 宣言（~11vw） | 200 (ExtraLight) | 超大字号用轻字重，避免压迫感 |
| 章节标题（~7vw） | 200 (ExtraLight) | 同上 |
| 正文标题（~2.6vw） | 300 (Light) | 中等字号用轻字重 |
| 正文（~1.05vw） | 400 (Regular) | 小字号用标准字重 |
| KPI 数字（~22vw） | 800 (ExtraBold) | 例外：数据数字需要视觉冲击 |
| 分类标签（11px） | 600 (SemiBold) | 小标签用重字重强调 |

**违反这个规则 = 失去瑞士风的灵魂。** 不要因为"觉得标题不够醒目"就加粗——瑞士风的醒目来自字号对比，不是字重。

---

## 第四阶段：色彩约束

### 单一强调色原则
- 一份 deck 只用一个 accent 色
- accent 色只用于：关键数据、按钮、色块填充、装饰线
- 不要满屏都是 accent——大面积 accent 只在封面/收尾页使用

### 灰阶层次
- `#0A0A0A`（ink）→ 主文字
- `#737373`（grey-3）→ 辅助文字/元数据
- `#D4D4D2`（grey-2）→ 分割线/边框
- `#F0F0EE`（grey-1）→ 区块底色
- `#FAFAF8`（paper）→ 主背景

### 深/浅交替
- 封面和收尾可以用 accent 全填充或 ink 黑底
- 正文页以纸色为主，穿插 ink 黑底反转页
- 禁止连续 3 页以上相同背景

---

## 第五阶段：间距与对齐

### 画布安全区
- 页面内容不要触及底部翻页点（底部留 ~8vh 安全区）
- Chrome 页眉和主体内容共用同一水平边线（不要二次叠加 padding）

### 间距控制
- 用 grid `gap` 控制间距，不用 margin/padding 堆叠
- 标题到内容区：gap 3-5vh
- 卡片之间：gap 4（16px）
- 组件内边距：p-5 或 p-6

### 对齐规则
- 所有内容左对齐（标题、正文、卡片）
- 禁止正文标题居中
- 多列内容顶部对齐（`items-start`）

---

## 第六阶段：质量自检

### P0（必须通过）
- 字重层次正确：大标题 200，正文 300-400，标签 600
- 画布尺寸正确：`w-[1280px] h-[720px]`
- 标题左对齐，不居中
- Kicker 在标题上方（上下结构）
- 不混用强调色（一份 deck 一个 accent）
- 不使用渐变、圆角、阴影
- 图片容器直角，用标准比例
- 使用 lucide-react 图标，不使用 emoji

### P1（布局节奏）
- 使用了至少 3 种不同的布局模式
- 不连续 3 页使用同一布局
- 深浅背景交替
- 8 页以上至少有 1 个封面 + 1 个收尾

### P2（视觉打磨）
- 点阵/装饰元素只在封面/收尾使用，正文页保持克制
- KPI 数字字号足够大（至少 8vw）
- 分割线只用 1px hairline
- 卡片填充类型不混用（ink/accent/grey/outlined 互斥）
