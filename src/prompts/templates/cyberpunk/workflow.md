# 赛博朋克设计方法论 (Cyberpunk Neon SOP)

---

## 第一阶段：数据提取

赛博朋克的核心是"数据即视觉"。面对用户输入，将其转化为终端化、网格化的数据呈现：

| 内容类型 | 数据化策略 | 视觉表达 |
|---------|-----------|---------|
| 核心观点 | 提取为"系统声明"或"终端输出" | 等宽字体 + 霓虹边框 |
| 数据指标 | 转化为 HUD 仪表盘 | 大数字 + 扫描线动画 |
| 流程/步骤 | 转化为系统节点图 | 网格连接 + 发光路径 |
| 对比/比较 | 转化为"协议 vs 协议" | 左右分屏 + 霓虹分割线 |
| 技术细节 | 转化为终端日志 | 等宽字体 + 暗色背景 |
| 图片展示 | 转化为监控画面/全息投影 | 扫描线叠加 + 霓虹边框 |
| 引用/金句 | 转化为系统广播 | 霓虹色边框 + 闪烁标点 |
| 章节过渡 | 转为系统启动/关机画面 | 全屏深色 + 扫描线 |

**核心原则：一切内容都是"系统数据"。用终端语言重写所有文案。**

## 第二阶段：霓虹布局

### 网格即骨架
赛博朋克的布局以网格为基础，所有元素必须对齐到网格线。

### 布局选择

| 布局模式 | 适用场景 | 实现要点 |
|---------|---------|---------|
| HUD 全屏 | 核心数据、开场 | 数据占据主体，四角放元数据 |
| 双栏终端 | 对比、并列 | 左右各 50%，中间霓虹分割线 |
| 三栏数据板 | 多维数据 | 三列等宽，顶部统一边框 |
| 监控矩阵 | 图片展示 | 2x2 或 3x2 网格，单元格等距 |
| 终端日志 | 技术细节 | 单列，左对齐，等宽字体 |
| 系统广播 | 引用、声明 | 居中，四周发光边框 |

### 网格间距
- 页面外边距：`p-6` 或 `p-8`
- 网格间距：`gap-4`（16px）
- 卡片内边距：`p-4` 或 `p-5`
- 元素之间用发光分割线（1px）分隔

## 第三阶段：发光效果系统

### 霓虹色彩体系

| 色彩角色 | 色值 | 使用场景 |
|---------|------|---------|
| 主背景 | `#0A0A0F`（深空黑） | 页面背景 |
| 次背景 | `#12121A`（暗紫黑） | 卡片、容器 |
| 主霓虹 | `#00F0FF`（赛博蓝） | 主强调色、边框、标题 |
| 次霓虹 | `#FF00AA`（霓虹粉） | 次强调色、数据高亮 |
| 警告色 | `#FFE600`（电光黄） | 警告、状态指示 |
| 主文字 | `#E0E0E8`（冷白） | 正文 |
| 辅助文字 | `#6A6A7A`（钢灰） | 注释、元数据 |

### 发光效果规则
- 霓虹边框：`border-1` + `box-shadow: 0 0 8px {color}, 0 0 16px {color}`
- 发光文字：`text-shadow: 0 0 8px {color}`
- 发光强度分级：
  - 弱：`0 0 4px`（辅助元素）
  - 中：`0 0 8px`（普通边框）
  - 强：`0 0 16px`（标题、关键数据）
- 每页发光元素不超过 5 个，避免视觉疲劳

### 背景效果
- 使用 CSS `repeating-linear-gradient` 创建扫描线效果
- 扫描线：`background: repeating-linear-gradient(0deg, transparent, transparent 2px, rgba(0,240,255,0.03) 2px, rgba(0,240,255,0.03) 4px)`
- 网格点阵：使用小圆点背景图案（间距 20px，大小 1px，颜色 `rgba(0,240,255,0.1)`）

## 第四阶段：终端排版

### 字体选择

| 元素 | 字体 | 字重 | 字号参考 |
|------|------|------|---------|
| 系统标题 | 等宽体（font-mono） | 700（Bold） | 3vw-5vw |
| 数据数字 | 等宽体（font-mono） | 800（ExtraBold） | 10vw-18vw |
| 终端正文 | 等宽体（font-mono） | 400（Regular） | 1vw-1.2vw |
| 标注文字 | 等宽体（font-mono） | 400（Regular） | 0.7vw |
| 辅助信息 | 无衬线体（font-sans） | 300（Light） | 0.8vw |

**关键规则：等宽字体是赛博朋克的灵魂。标题、数字、正文全部使用等宽体。**

### 文字效果
- 标题全大写（`uppercase`），字间距加宽（`tracking-widest`）
- 数据数字添加霓虹色
- 终端文字使用 `>` 前缀或 `//` 注释前缀
- 关键词用霓虹色高亮

### 动态元素（CSS 实现）
- 扫描线动画：`@keyframes scanline` 从上到下移动
- 文字闪烁：`@keyframes flicker` 随机透明度变化
- 数据滚动：终端日志自动向上滚动效果

## 第五阶段：元素规范

### 容器样式
- 卡片背景：`bg-[#12121A]`
- 边框：`border border-[#00F0FF]/30`（半透明霓虹边）
- 圆角：`rounded-none`（直角，赛博朋克不用圆角）
- 阴影：霓虹发光阴影，不用普通阴影

### 允许的元素
- 文字（等宽体为主）
- 霓虹边框容器
- 发光分割线（1px，带 glow）
- 网格点阵背景
- 扫描线叠加
- lucide-react 图标（配合霓虹色）

### 禁止的元素
- 圆角（`rounded-*`）
- 柔和渐变
- 暖色调
- emoji
- 纸质/自然纹理
- 阴影（用发光效果代替）

### 图片处理
- 图片加扫描线叠加层
- 图片边框使用霓虹色
- 图片比例统一（16:9）
- 可选：加轻微色相偏移（偏蓝/偏紫）

## 第六阶段：冲击力自检

### P0（必须通过）
- 画布尺寸正确：`w-[1280px] h-[720px]`（16:9）或 `w-[1024px] h-[768px]`（4:3）
- 深色背景为主（黑色/深紫占比 80% 以上）
- 等宽字体使用率 70% 以上
- 至少有 1 个霓虹发光效果（边框或文字）
- 无圆角、无暖色、无柔和渐变
- 使用 lucide-react 图标，不使用 emoji

### P1（视觉节奏）
- 霓虹色使用一致（主色/次色不混用超过 2 种）
- 发光强度有层次（强/中/弱）
- 不连续 3 页使用相同布局
- 8 页以上 deck 至少有 1 个 HUD 全屏 + 1 个终端日志页

### P2（细节打磨）
- 扫描线效果存在但不喧宾夺主（透明度 < 5%）
- 霓虹发光不溢出画布边界
- 文字对比度达标（冷白 `#E0E0E8` on `#0A0A0F`）
- 辅助文字足够淡（`#6A6A7A`），不抢主视觉
- 数据数字字号足够大（至少 8vw），有视觉冲击力
