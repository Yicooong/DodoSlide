# 核心操作指令

## 1. 风格约束原则 (Procedure)
- **画布比例**：严格锁定为 1280x720 (16:9)。
- **语气定位**：保持学术严谨性，禁止使用过于跳跃或高饱和度的装饰。
- **视觉重心**：遵循“数据/图表为主，解释为辅”的原则。

## 2. 布局模式选择 (Workflow)
根据内容特征，从以下三类模式中选择其一执行：
- **逻辑映射类**：用于解释“成因 -> 机制 -> 修复”。采用水平卡片流，使用 `flex items-stretch` 确保视觉齐平。
- **架构流转类**：用于展示系统 Pipeline。采用“灰色原生流”与“彩色优化组件”对比。
- **数据看板类**：用于实验指标。左侧放置 Big Number 或进度条，右侧进行归因分析。

## 3. 反模式清单 (Anti-Patterns)
**严禁出现以下行为：**
- **色彩过载**：禁止在单页使用超过 3 种明亮色相。
- **空间拥挤**：单页内边距 (Padding) 不得小于 `p-8`。
- **公式裸奔**：禁止在 JSX 中直接书写带大括号的 LaTeX，必须使用字符串包装：`{"$$公式$$"}` 或 HTML 组件模拟。
- **术语混淆**：严格区分“图中中断修复率”与“转换率”，前者侧重于修复动作的有效性。
